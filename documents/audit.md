# Аудит TypeScript проекта

Проанализируй код проекта по следующим критериям:

## 1. Дублирование и повторы
- Идентичный или похожий код в разных местах
- Функции/методы с разными именами, выполняющие одинаковую логику
- Дублирующиеся типы, интерфейсы, утилиты
- Повторяющиеся паттерны обработки данных
- Копипаста компонентов/модулей

## 2. Архитектура и структура
- Нарушения принципов SOLID
- Циклические зависимости между модулями
- Избыточная связанность (tight coupling)
- Отсутствие слоев абстракции где нужно
- Неоптимальная структура папок/модулей

## 3. Производительность
- Лишние ререндеры (React/Vue/Angular)
- Отсутствие мемоизации где она нужна
- Неэффективные алгоритмы (O(n²) вместо O(n))
- Избыточные циклы и итерации
- Синхронный код, который можно распараллелить
- Отсутствие lazy loading и code splitting
- Неоптимальная работа с DOM

## 4. Типизация
- any вместо конкретных типов
- Отсутствующие или слабые типы
- Игнорирование ошибок TypeScript через @ts-ignore
- Дублирование типов

## 5. Обработка данных
- Мутации вместо иммутабельности
- Лишние трансформации данных
- Неоптимальная работа с массивами (map → filter → reduce можно объединить)
- Отсутствие кеширования частых вычислений

## 6. Асинхронность
- Отсутствие обработки ошибок в промисах
- Последовательные await, которые можно выполнить параллельно
- Memory leaks из-за незавершенных подписок
- Отсутствие debounce/throttle где нужно

## 7. Импорты и бандл
- Импорт всей библиотеки вместо tree-shaking
- Неиспользуемые импорты и зависимости
- Большие чанки, которые можно разбить

## 8. Код-смеллы
- Длинные функции (>50 строк)
- Глубокая вложенность (>3-4 уровня)
- Магические числа и строки
- Богатые комментарии вместо рефакторинга
- Неиспользуемый код (dead code)

Для каждой найденной проблемы укажи:
- Местоположение в коде
- Описание проблемы
- Предложение по улучшению/исправлению/реализации

Результат в tsProblems.md под найденные проблемы пиши чек-лист задач для решения проблем
